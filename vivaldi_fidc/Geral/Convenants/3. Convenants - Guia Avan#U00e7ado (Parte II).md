## **12. STRESS TESTING E ANÁLISE DE CENÁRIOS**

### **12.1 Fundamentos Metodológicos do Stress Testing**

**Definição Conceitual:**

Stress testing é uma técnica de simulação que avalia como a carteira de um FIDC e seus covenants se comportariam sob condições adversas extremas mas plausíveis.

**Objetivos do Stress Testing em FIDCs:**

1. **Avaliar Robustez dos Covenants**: Verificar se limites estabelecidos são suficientes para cenários extremos
2. **Quantificar Perdas Potenciais**: Estimar impacto financeiro em cada classe de cotas
3. **Identificar Vulnerabilidades**: Detectar concentrações de risco não aparentes em análise normal
4. **Subsidiar Pricing**: Informar taxa adequada de remuneração das cotas
5. **Atender Regulação**: Cumprir exigências de gestão de riscos (Resolução CVM 175)

### **12.2 Tipos de Cenários de Estresse**

**a) Cenários Históricos (Historical Scenarios):**

Replicam eventos reais passados

**Exemplo: Crise COVID-19 (Mar-Jun/2020)**

```
Parâmetros Observados:
- Inadimplência 90+ aumentou de 2,5% para 8,3% (+230%)
- Taxa de recuperação caiu de 70% para 45%
- Prazo médio de cobrança aumentou 90 → 150 dias
- 15% dos devedores solicitaram renegociação

Aplicação ao FIDC:
Carteira Base = R$ 100MM
Inadimpl. Normal = 2,5% = R$ 2,5MM
Inadimpl. Stress = 8,3% = R$ 8,3MM
Aumento = R$ 5,8MM

Impacto em Covenants:
Covenant: Inadimpl 90+ ≤ 3,5%
Stress: 8,3%
Violação: 4,8 p.p.

Provisão Necessária:
LGD stress = 55% (1 - 0,45)
Provisão = R$ 8,3MM × 55% = R$ 4,57MM

Subordinação Necessária:
Atual: 30% = R$ 30MM
Consumida por perdas: R$ 4,57MM
Remanescente: R$ 25,43MM (25,43%)
Status: Cotas seniores protegidas, mas margem reduzida
```

**Exemplo: Crise 2008 (Set/2008 - Dez/2009)**

```
Parâmetros:
- Inadimplência disparou de 3,0% para 12,5%
- Spread de crédito: +650 bps
- Liquidez: Mercado secundário congelou
- Recompras forçadas: 25% da carteira

Simulação FIDC Imobiliário:
Carteira: Financiamentos habitacionais
LTV médio: 70%
Taxa inadimpl. normal: 1,8%
Taxa inadimpl. stress: 7,5%

Perdas Estimadas:
Inadimpl = R$ 100MM × 7,5% = R$ 7,5MM
Valor imóveis caiu 30% → LGD aumentou
LGD normal = 25% (LTV 70%, margem 30%)
LGD stress = 55% (valor imóvel caiu, custos execução altos)
Perda = R$ 7,5MM × 55% = R$ 4,125MM

Covenant DL/EBITDA Originador:
Normal: 2,5x
Stress: 4,2x (EBITDA caiu 40%)
Limite: 3,5x → VIOLAÇÃO
```

**b) Cenários Hipotéticos (Hypothetical Scenarios):**

Construídos com base em riscos potenciais identificados

**Cenário: Recessão Severa Brasil 20XX**

```
Premissas:
- PIB: -5,0% a.a.
- Inflação: 12% a.a.
- Desemprego: 15% (vs. 8% normal)
- Selic: 15% a.a.
- Câmbio: R$/US$ 7,00 (desvalorização 40%)

Impactos na Carteira FIDC Varejo:
1. Inadimplência:
   β_inadimpl_PIB = -2,5 (elasticidade)
   Δ Inadimpl = -2,5 × (-5%) = +12,5 p.p.
   Nova inadimpl = 3% + 12,5% = 15,5%

2. Taxa de Recuperação:
   Desemprego alto → menor capacidade pagamento
   Taxa recup. normal = 65%
   Taxa recup. stress = 40%
   LGD normal = 35%
   LGD stress = 60%

3. Velocidade de Cobrança:
   Prazo médio normal = 90 dias
   Prazo médio stress = 180 dias
   Impacto liquidez: duplicação do gap

4. Concentração Setorial:
   Setor mais afetado: Comércio (-8% PIB setorial)
   Se FIDC tem 40% exposição ao comércio:
   Inadimpl. comércio = 20% (vs. 15,5% média)
```

**Cálculo de Impacto Total:**

```
Perda Esperada Base:
EL_base = PD × LGD × EAD
EL_base = 3% × 35% × R$ 100MM = R$ 1,05MM

Perda Esperada Stress:
EL_stress = 15,5% × 60% × R$ 100MM = R$ 9,3MM

Delta = R$ 8,25MM

Comparação com Proteções:
Subordinação: R$ 30MM ✓ Suficiente
Provisão: R$ 2MM ✗ Insuficiente
Gap = R$ 7,3MM → Necessário reforço
```

**c) Cenários Reversos (Reverse Stress Testing):**

Identificam condições que levariam à quebra do FIDC

**Metodologia:**

```
Pergunta: Qual nível de inadimplência esgotaria toda subordinação?

Dados:
Subordinação = R$ 30MM (30% de R$ 100MM)
Provisão = R$ 2MM
Total proteção = R$ 32MM

Carteira = R$ 100MM
LGD assumida = 50%

Inadimpl_crítica = Proteção / (Carteira × LGD)
Inadimpl_crítica = R$ 32MM / (R$ 100MM × 0,50)
Inadimpl_crítica = 64%

Interpretação: 
Se 64% da carteira se tornar inadimplente com LGD de 50%,
cotas seniores começarão a ter perdas.

Questão: Quão plausível é 64% de inadimplência?
- Nunca ocorreu em FIDCs brasileiros (máx histórico: ~25% em 2015)
- Requer evento catastrófico sistêmico
- Portanto, proteção é ROBUSTA para cenários plausíveis
```

**d) Cenários Multifatoriais Correlacionados:**

Combinam múltiplos choques simultâneos

**Exemplo: Tripla Crise**

```
Fator 1: Econômico (Recessão)
  - PIB: -4%
  - Desemprego: +5 p.p.

Fator 2: Setorial (Crise no Agronegócio)
  - Preço commodities: -40%
  - Inadimpl. agro: +15 p.p.

Fator 3: Originador (Deterioração Financeira)
  - DL/EBITDA: 2,5x → 5,0x
  - Rating: downgrade 3 notches
  - Custo cobrança: +50%

FIDC Exposto:
- 35% Agronegócio
- 65% Outros setores
- Originador é servicer exclusivo

Impactos Combinados:
Inadimpl_agro = 3% + 15% = 18%
Inadimpl_outros = 3% × (1 + β × ΔPIB)
                = 3% × (1 + 2 × 0,04) = 3,24%

Inadimpl_total_ponderada:
= 0,35 × 18% + 0,65 × 3,24%
= 6,3% + 2,11% = 8,41%

LGD aumenta devido ineficiência cobrança:
LGD_normal = 35%
LGD_stress = 35% × 1,3 (custo +30%) = 45,5%

Perda = R$ 100MM × 8,41% × 45,5% = R$ 3,83MM

Covenants Afetados:
1. Inadimpl 90+ ≤ 3,5% → 8,41% VIOLAÇÃO
2. DL/EBITDA orig ≤ 3,5x → 5,0x VIOLAÇÃO
3. Concentração agro ≤ 30% → 35% VIOLAÇÃO (já existente)

Conclusão: Cenário CRÍTICO, múltiplas violações simultâneas
```

### **12.3 Métricas de Stress Testing**

**a) Perda Inesperada (Unexpected Loss):**

```
UL = √(Var[Perdas])

Onde Var[Perdas] considera:
- Volatilidade da PD
- Volatilidade da LGD
- Correlação entre defaults

Fórmula Detalhada (Basel II):
UL = EAD × √[PD × σ²_LGD + LGD² × σ²_PD]

Exemplo:
EAD = R$ 100MM
PD = 5%
LGD = 40%
σ_PD = 3% (desvio-padrão)
σ_LGD = 15%

UL = 100MM × √[0,05 × 0,15² + 0,40² × 0,03²]
UL = 100MM × √[0,001125 + 0,000144]
UL = 100MM × √0,001269
UL = 100MM × 0,0356
UL = R$ 3,56MM

Perda em Nível de Confiança 99,9%:
Perda_999 = EL + 3,09 × UL
          = R$ 2MM + 3,09 × R$ 3,56MM
          = R$ 13,0MM

Verificação Adequação Capital:
Subordinação = R$ 30MM
Perda_999 = R$ 13MM
Margem = R$ 17MM ✓ ADEQUADO
```

**b) Conditional Value at Risk (CVaR):**

Também conhecido como Expected Shortfall

```
CVaR_α = E[Perda | Perda > VaR_α]

Significado: Perda média dado que estamos na cauda α

Exemplo:
Distribuição de perdas em stress (10.000 simulações):
VaR_95% = R$ 8MM (95ª pior perda)

Perdas piores que R$ 8MM:
R$ 8,2MM, R$ 8,5MM, R$ 9,1MM, ..., R$ 15,3MM (500 cenários)

CVaR_95% = Média dessas 500 piores perdas
         = R$ 10,5MM

Interpretação:
- Se as coisas ficarem muito ruins (5% piores casos)
- Perda média esperada = R$ 10,5MM
- Subordinação R$ 30MM ainda cobre com margem
```

**c) Stress Coverage Ratio (SCR):**

```
SCR = Reforço_Crédito_Total / Perda_Stress_Cenário

Exemplo:
Reforço Crédito:
- Subordinação: R$ 30MM
- Reserva caixa: R$ 3MM
- Excesso spread acumulado: R$ 2MM
Total = R$ 35MM

Perda Stress (pior cenário):
= R$ 12MM

SCR = R$ 35MM / R$ 12MM = 2,92x

Interpretação:
Proteção é 2,92 vezes a perda no pior cenário testado
Rating esperado: AA+ a AAA (tipicamente SCR > 2,5x)
```

### **12.4 Implementação Prática: Simulação Monte Carlo**

**Algoritmo:**

```python
Copyimport numpy as np

def stress_test_fidc(carteira, n_simul=10000):
    """
    Stress test via Monte Carlo
    """
    resultados = []
    
    for i in range(n_simul):
        # Simular choques macroeconômicos
        pib_shock = np.random.normal(-2, 1.5)  # média -2%, σ 1,5%
        desemprego_shock = np.random.normal(2, 1)  # média +2p.p.
        
        # Impacto na inadimplência
        beta_pib = -2.5
        beta_desemp = 0.8
        pd_base = 0.03  # 3%
        
        pd_stress = pd_base * (1 + beta_pib * pib_shock/100 + 
                              beta_desemp * desemprego_shock/100)
        pd_stress = max(0, min(pd_stress, 1))  # Limitar [0,1]
        
        # Simular LGD correlacionado
        lgd_base = 0.35
        lgd_stress = lgd_base + np.random.normal(0.15, 0.08)
        lgd_stress = max(0, min(lgd_stress, 1))
        
        # Calcular perda
        perda = carteira * pd_stress * lgd_stress
        
        resultados.append({
            'pib': pib_shock,
            'desemprego': desemprego_shock,
            'pd': pd_stress,
            'lgd': lgd_stress,
            'perda': perda
        })
    
    return pd.DataFrame(resultados)

# Executar
carteira = 100_000_000  # R$ 100MM
df_stress = stress_test_fidc(carteira, n_simul=10000)

# Estatísticas
print(f"Perda Média: R$ {df_stress['perda'].mean():,.0f}")
print(f"VaR 95%: R$ {df_stress['perda'].quantile(0.95):,.0f}")
print(f"VaR 99%: R$ {df_stress['perda'].quantile(0.99):,.0f}")
print(f"CVaR 95%: R$ {df_stress[df_stress['perda'] > df_stress['perda'].quantile(0.95)]['perda'].mean():,.0f}")
```

**Saída Exemplo:**

```
Perda Média: R$ 2.145.000
VaR 95%: R$ 7.800.000
VaR 99%: R$ 11.200.000
CVaR 95%: R$ 9.450.000

Cenário Pior (Percentil 99,9%): R$ 14.500.000

Análise de Covenants sob Stress:
┌────────────────────────────┬──────────┬───────────┬──────────┐
│ Covenant                   │ Normal   │ VaR 99%   │ Status   │
├────────────────────────────┼──────────┼───────────┼──────────┤
│ Inadimpl 90+ ≤ 3,5%        │ 2,8%     │ 11,2%     │ VIOLA    │
│ Provisão ≥ 2% PL           │ 2,1%     │ 11,2%     │ VIOLA    │
│ Subordinação ≥ 30%         │ 30%      │ 18,8%*    │ VIOLA    │
└────────────────────────────┴──────────┴───────────┴──────────┘

* Subordinação consumida por perdas: 30% - 11,2% = 18,8%

Recomendação: Aumentar subordinação para 35% ou adicionar
              garantias externas de R$ 5MM
```

## **13. RATING DE CRÉDITO E METODOLOGIAS DAS AGÊNCIAS**

### **13.1 Fundamentos do Rating em FIDCs**

**Definição:**

Rating é uma opinião independente sobre a capacidade de um FIDC honrar pagamentos das cotas de acordo com prazos e montantes previstos no regulamento.

**Principais Agências no Brasil:**

1. **Fitch Ratings**
2. **Moody's Local (antiga SR Rating)**
3. **Standard & Poor's (S&P)**
4. **DBRS Morningstar**

**Escala Nacional de Rating (Brasil):**

```
┌─────────────┬──────────────────────────────────────────────────┐
│ Rating      │ Significado                                      │
├─────────────┼──────────────────────────────────────────────────┤
│ AAA(bra)    │ Mais alta qualidade, risco mínimo               │
│ AA(bra)     │ Qualidade muito alta                             │
│ A(bra)      │ Qualidade alta, vulnerabilidade moderada        │
│ BBB(bra)    │ Qualidade média, alguma vulnerabilidade         │
│ BB(bra)     │ Especulativo, vulnerabilidade significativa     │
│ B(bra)      │ Altamente especulativo                           │
│ CCC(bra)    │ Risco substancial, possibilidade real default  │
│ CC(bra)     │ Default provável                                 │
│ C(bra)      │ Default ou equivalente                           │
│ D(bra)      │ Default efetivo                                  │
└─────────────┴──────────────────────────────────────────────────┘

Modificadores: + ou - (dentro de cada categoria AA a CCC)
Exemplo: AA+(bra), AA(bra), AA-(bra)
```

### **13.2 Metodologia Fitch para FIDCs**

**Fatores de Rating (Pesos Relativos):**

```

   c) Histórico de inadimplência
   d) Setor econômico

2. Estrutura Jurídica e Segregação (20%)
   a) Robustez da cessão (true sale)
   b) Qualidade da documentação
   c) Registro e custódia
   d) Proteção contra recuperação judicial originador

3. Reforço de Crédito (25%)
   a) Subordinação
   b) Excesso de spread
   c) Reservas de caixa
   d) Garantias externas

4. Administração e Gestão (15%)
   a) Experiência do gestor
   b) Sistemas de controle
   c) Histórico de performance
   d) Governança

5. Covenant Framework (5%)
   a) Rigor dos covenants
   b) Mecanismos de enforcement
   c) Gatilhos e remédios
```

**Análise Quantitativa Detalhada:**

**a) Cálculo de Subordinação Necessária:**

## **14. CROSS-DEFAULT E ACELERAÇÃO CRUZADA**

### **14.1 Conceito e Racionalidade Econômica**

**Definição Jurídica:**

**Cross-Default** (Inadimplência Cruzada) é uma cláusula contratual que estabelece que o inadimplemento de uma obrigação em um contrato específico constitui automaticamente evento de inadimplemento em outros contratos relacionados.

**Fundamentação Teórica:**

```
Problema sem Cross-Default:
Empresa emite:
- Debênture A: R$ 50MM
- FIDC B: R$ 30MM  
- Debênture C: R$ 40MM

Cenário:
Empresa tem crise, não paga Debênture A

Sem Cross-Default:
- Debenturistas A executam garantias
- Empresa entra em recuperação judicial
- FID C B e Debenturistas C ficam subordinados na fila

Com Cross-Default:
- Não pagamento de A → trigger em B e C
- Todos aceleram simultaneamente (pari passu)
- Rateio proporcional dos ativos
- Protege credores que detectam problema mais tarde
```

**Tipos de Cross-Default:**

**a) Cross-Default Puro:**

```
Inadimplemento em qualquer dívida → Inadimplemento em todas
```

**b) Cross-Default Qualificado (Material):**

```
Apenas inadimplementos "materiais" acionam cross-default

Definição de Material:
- Valor > R$ X milhões (threshold)
- Duração > Y dias
- Após período de cura (cure period)

Exemplo:
"Constitui Evento de Inadimplemento o descumprimento pelo Originador
de qualquer obrigação financeira superior a R$ 5.000.000 (cinco
milhões de reais) que não seja sanado em 10 (dez) dias úteis."
```

**c) Cross-Acceleration (Aceleração Cruzada):**

```
Vencimento antecipado em qualquer dívida → Vencimento antecipado aqui

Diferença sutil mas importante:
- Cross-Default: inadimplemento → inadimplemento
- Cross-Acceleration: vencimento antecipado → vencimento antecipado

Cross-Acceleration é mais amplo (inclui defaults curados com waiver)
```

### **14.2 Estruturação de Cross-Default em FIDCs**

**Modelo de Cláusula:**

```
Cláusula X - Evento de Inadimplemento por Cross-Default

Constituirá Evento de Inadimplemento a ocorrência de qualquer das
seguintes hipóteses em relação ao Originador:

a) Inadimplemento de qualquer Dívida Financeira em valor individual
   ou agregado superior a R$ [THRESHOLD] que perdure por mais de
   [DIAS] dias corridos contados da data de vencimento;

b) Vencimento antecipado declarado em relação a qualquer Dívida
   Financeira em valor individual ou agregado superior a
   R$ [THRESHOLD];

c) Não pagamento, no vencimento, de principal ou juros de qualquer
   título ou valor mobiliário emitido pelo Originador (debêntures,
   notas comerciais, CRA, CRI) em valor superior a R$ [THRESHOLD];

d) Protesto de títulos em valor agregado superior a R$ [THRESHOLD]
   que não sejam cancelados ou suspensos em [DIAS] dias;

e) Decretação de falência, pedido de recuperação judicial ou
   extrajudicial, liquidação extrajudicial, ou início de processo
   de insolvência.

Para fins desta cláusula:

"Dívida Financeira" significa qualquer endividamento junto a
instituições financeiras, mercado de capitais (debêntures, notas
promissórias) ou FIDCs, excluindo:
  (i) operações de leasing operacional
  (ii) dívidas tributárias em discussão administrativa/judicial
  (iii) fornecedores no curso normal dos negócios

"Threshold" (Valor Mínimo de Materialidade):
- Para cross-default de dívidas: R$ 10.000.000
- Para protestos: R$ 5.000.000
- Para valores mobiliários: Qualquer valor

"Cure Period" (Período de Cura):
- Dívidas bancárias: 15 dias úteis
- Valores mobiliários: 5 dias úteis
- Protestos: 10 dias corridos
```

**Parâmetros de Calibragem:**

```
Threshold (Valor Mínimo):
Muito Baixo (R$ 500k): 
  ✓ Máxima proteção
  ✗ Risco de triggers espúrios
  
Balanceado (R$ 5-10MM):
  ✓ Filtra inadimplementos triviais
  ✓ Captura problemas materiais
  ✓ Mercado padrão
  
Muito Alto (R$ 50MM):
  ✓ Evita acelerações prematuras
  ✗ Pode ser tarde demais quando aciona

Recomendação:
Threshold = 5% a 10% do Patrimônio Líquido do Originador

Exemplo:
PL Originador = R$ 100MM
Threshold = 7% × R$ 100MM = R$ 7MM
```

**Cure Period (Período de Cura):**

```
Sem Cure Period:
  ✗ Atraso de 1 dia → Default imediato
  ✗ Problemas operacionais viram crises
  
Cure Period Curto (3-5 dias):
  ✓ Proteção rápida
  ✗ Pouco tempo para resolver
  
Cure Period Padrão (10-15 dias úteis):
  ✓ Tempo razoável para regularizar
  ✓ Distingue problema técnico de crise
  ✓ Mercado padrão
  
Cure Period Longo (30 dias):
  ✓ Máxima flexibilidade
  ✗ Pode ser tarde demais
  
Recomendação:
- Valores mobiliários: 5-10 dias (mercado público, reputação)
- Dívidas bancárias: 10-15 dias
- Fornecedores: 30 dias (curso normal)
```

**Remédios Disponíveis aos Cotistas:**

```
1. Vencimento Antecipado (Acceleration):
   Quorum: 2/3 das cotas seniores
   Efeito: FIDC entra em liquidação imediata
   Processo:
   - Suspensão de todas operações
   - Cobrança acelerada de todos DC
   - Venda de ativos líquidos
   - Distribuição proporcional (waterfall)

2. Execução de Garantias:
   Se Originador deu garantias:
   - Carta fiança bancária: Saque imediato
   - Penhor de ações: Venda compulsória
   - Cessão fiduciária: Apropriação recebíveis

3. Substituição Servicer:
   Se Originador é servicer:
   Quorum: Maioria simples cotas seniores
   Novo servicer contratado (fee ~2-3% recebíveis)

4. Liquidação Judicial:
   Último recurso
   Ação de execução de título extrajudicial
   Cotas representam crédito quirografário

5. Waiver Condicional:
   Alternativa à aceleração
   Condições típicas:
   - Pagamento waiver fee (1-3%)
   - Reforço garantias
   - Aumento subordinação
   - Restrições operacionais originador
```

### **14.4 Casos Práticos e Jurisprudência**

**Caso 1: Recuperação Judicial e Cross-Default**

```
Situação:
Originador entrou em recuperação judicial
FIDC tinha cross-default para "pedido RJ"

Questão Jurídica:
Cross-default é acionável automaticamente ou precisa assembleia?

TJ-SP (Apelação nº 1234567-89.2020):
"A cláusula de cross-default, por si só, não torna exigível 
antecipadamente a dívida. É necessário que o credor (cotistas)
deliberem em assembleia sobre exercer ou não a faculdade de
considerar vencida antecipadamente a obrigação."

Aprendizado:
Cross-default cria DIREITO de acelerar, não obrigação automática
Regulamento deve prever claramente: assembleia decide ou é automático

Redação Recomendada:
"Configurado Evento de Cross-Default, o Administrador PODERÁ,
independente de assembleia, declarar vencidas antecipadamente
todas as obrigações, EXCETO se assembleia de cotistas deliberar
em contrário em até 15 dias."
(Opt-out ao invés de opt-in)
```

**Caso 2: Cross-Default de Protestos**

```
Situação:
Empresa teve R$ 8MM em protestos (diversos fornecedores)
FIDC tinha threshold R$ 5MM para protestos
Cross-default acionado

Defesa do Originador:
"Protestos são de boa-fé, referem-se a disputas comerciais
legítimas em discussão judicial. Não representam crise financeira."

TJ-SP (Agravo de Instrumento nº 9876543-21.2021):
"O mero protesto, desacompanhado de outros elementos que indiquem
deterioração da situação econômico-financeira, não justifica,
por si só, o vencimento antecipado das obrigações."

Aprendizado:
Cross-default de protestos deve ter qualificadores:
- Protestos não cancelados após X dias
- Desde que não suspensa a exigibilidade por decisão judicial
- Que não estejam garantidos

Redação Aprimorada:
"Protestos em valor superior a R$ X que não sejam (i) cancelados,
(ii) suspensos por decisão judicial, ou (iii) garantidos por
seguro ou depósito judicial, no prazo de 30 dias corridos."
```

## **15. BACKTESTING E VALIDAÇÃO DE MODELOS**

### **15.1 Fundamentos Estatísticos**

**Objetivo do Backtesting:**

Verificar se modelos de PD e LGD utilizados para:

- Precificação de cotas
- Cálculo de provisões
- Definição de covenants

São **estatisticamente acurados** quando comparados com resultados realizados.

**Hipótese Nula:**

```
H₀: Modelo está calibrado corretamente
     (defaults observados = defaults previstos)

H₁: Modelo está mal calibrado
     (discrepância estatisticamente significativa)
```

### **15.2 Teste de Binomial (Backtesting de PD)**

**Metodologia:**

```
Dados:
n = número de devedores na amostra
k = número de defaults observados
p = PD prevista pelo modelo

Sob H₀, k ~ Binomial(n, p)

Teste:
Calcular probabilidade de observar k ou mais defaults dado p:

P-valor = P(X ≥ k | n, p) = Σ(i=k até n) [C(n,i) × p^i × (1-p)^(n-i)]

Decisão:
Se P-valor < α (ex: 5%), rejeitar H₀ → Modelo está mal calibrado
```

**Exemplo Prático:**

```
FIDC com carteira de 500 devedores
Modelo prevê PD = 3% para cada devedor
Portanto, defaults esperados = 500 × 3% = 15

Após 12 meses, observou-se:
k = 28 defaults (87% a mais que previsto)

Teste Binomial:
n = 500
p = 0,03
k = 28

P-valor = P(X ≥ 28 | n=500, p=0,03)

Usando aproximação normal (n grande):
μ = np = 15
σ = √(np(1-p)) = √(14,55) = 3,81

Z = (28 - 15) / 3,81 = 3,41

P-valor = P(Z > 3,41) = 0,0003 (0,03%)

Conclusão:
P-valor = 0,03% << 5% → REJEITAR H₀
Modelo está SIGNIFICATIVAMENTE subestimando risco
Ação: Recalibrar modelo, aumentar PD ou aumentar provisões
```

### **15.3 Teste de Hosmer-Lemeshow**

Para modelos com PD variando por devedor:

```
Metodologia:
1. Ordenar devedores por PD crescente
2. Dividir em 10 decis (ou quantis)
3. Para cada decil:
   - Calcular defaults esperados: E_j = Σ PD_i
   - Observar defaults reais: O_j
4. Estatística de teste:

HL = Σ(j=1 até 10) [(O_j - E_j)² / E_j]

Sob H₀: HL ~ χ²(8) [qui-quadrado com 8 g.l.]

Decisão:
Se HL > χ²_crítico(8, 0,05) = 15,51 → Rejeitar H₀
```

**Exemplo:**

```
Backtesting de modelo PD - FIDC Agronegócio

┌────────┬──────────┬────────────┬────────────┬───────────┐
│ Decil  │ PD Média │ Esperados  │ Observados │ (O-E)²/E  │
├────────┼──────────┼────────────┼────────────┼───────────┤
│ 1      │ 0,5%     │ 5,0        │ 4          │ 0,20      │
│ 2      │ 1,2%     │ 12,0       │ 15         │ 0,75      │
│ 3      │ 1,8%     │ 18,0       │ 17         │ 0,06      │
│ 4      │ 2,5%     │ 25,0       │ 29         │ 0,64      │
│ 5      │ 3,2%     │ 32,0       │ 38         │ 1,13      │
│ 6      │ 4,1%     │ 41,0       │ 47         │ 0,88      │
│ 7      │ 5,3%     │ 53,0       │ 61         │ 1,21      │
│ 8      │ 7,0%     │ 70,0       │ 82         │ 2,06      │
│ 9      │ 9,5%     │ 95,0       │ 108        │ 1,78      │
│ 10     │ 15,2%    │ 152,0      │ 175        │ 3,48      │
├────────┼──────────┼────────────┼────────────┼───────────┤
│ TOTAL  │          │ 503,0      │ 576        │ 12,19     │

HL = 12,19

χ²_crítico(8, 0,05) = 15,51

12,19 < 15,51 → NÃO REJEITAR H₀

Conclusão:
Modelo tem calibração adequada (P-valor ≈ 14%)
Porém, nota-se subestimação nos decis superiores (maior risco)
Recomendação: Ajustar coeficientes para altos riscos
```

### **15.4 Teste de Kolmogorov-Smirnov (LGD)**

Para validar distribuição de LGD prevista vs. realizada:

```
Hipótese:
H₀: LGD realizada segue distribuição prevista

Estatística:
D = max|F_empírica(x) - F_modelo(x)|

Onde:
F_empírica = Função distribuição acumulada observada
F_modelo = Função distribuição acumulada do modelo

Valor crítico (α=5%, n=100):
D_crítico = 1,36 / √n = 1,36 / 10 = 0,136

Decisão:
Se D > 0,136 → Rejeitar H₀
```

**Exemplo:**

```
Backtesting LGD - FIDC Financiamentos Veiculares

Modelo prevê: LGD ~ Beta(α=2, β=5)
Média = α/(α+β) = 28,6%

100 defaults observados:
LGD realizadas ordenadas:
5%, 8%, 12%, 15%, 18%, ..., 75%, 82%

Comparação:
┌────────┬────────────────┬─────────────────┬───────────┐
│ LGD    │ F_empírica(x)  │ F_modelo(x)     │ |Dif|     │
├────────┼────────────────┼─────────────────┼───────────┤
│ 10%    │ 0,15           │ 0,08            │ 0,07      │
│ 20%    │ 0,32           │ 0,28            │ 0,04      │
│ 30%    │ 0,51           │ 0,52            │ 0,01      │
│ 40%    │ 0,69           │ 0,72            │ 0,03      │
│ 50%    │ 0,83           │ 0,86            │ 0,03      │
│ 60%    │ 0,92           │ 0,94            │ 0,02      │
│ 70%    │ 0,97           │ 0,98            │ 0,01      │
│ 80%    │ 0,99           │ 0,995           │ 0,005     │
└────────┴────────────────┴─────────────────┴───────────┘

D_max = 0,07 (em LGD=10%)

D_crítico = 0,136

0,07 < 0,136 → NÃO REJEITAR H₀

Conclusão:
Distribuição de LGD está bem calibrada
Leve subestimação em baixas LGDs (10%), mas não significativa
```
