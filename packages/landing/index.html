OS 9 AGENTES

Cada agente é um arquivo .md com personalidade,
competências e limites

Não são prompts genéricos. São SOULs — arquivos Markdown versionados que definem role, system_prompt, domains, keywords e capabilities. O AgentRegistry carrega todos de um diretório. O AgentDispatcher roteia por similaridade semântica.

# packages/agents/souls/compliance.md slug: compliance name: Agente de Compliance role: Verificador regulatório do FIDC domains: [regulatory, compliance, cvm, bacen] capabilities: - check_eligibility(receivable) → PASS | REJECT - check_concentration(portfolio, op) → PASS | WARN | REJECT - check_covenant(fund_state) → PASS | BREACH - check_pld_aml(entity) → PASS | FLAG | REJECT - check_compliance(operation) → PASS | REJECT delegates: check_risk → risk-agent calculate_pdd → pricing-agent

6 GUARDRAIL GATES

Toda operação passa por 6 verificações
antes de ser aprovada

Implementado em Python como GuardrailPipeline. Cada gate retorna PASS, WARN ou REJECT. Um REJECT em qualquer gate bloqueia a operação inteira.

# packs/finance/guardrails/compliance.py class GuardrailPipeline: def check(self, query, response, chunks, confidence) -> GuardrailResult: gates = [ self._gate_eligibility(response), # tipo, prazo, rating self._gate_concentration(response), # cedente ≤15%, sacado ≤10% self._gate_covenant(response), # liquidez, subordinação, inadimplência self._gate_pld_aml(query, response), # OFAC, PEP, transações atípicas self._gate_compliance(response, chunks), # CVM 175, regulamento self._gate_risk(response, confidence), # PDD projetada, stress test ] return GuardrailResult( passed=all(g.passed for g in gates), gates=gates )

COGNITIVE KERNEL

Não é um chatbot com RAG.
É um sistema cognitivo com routing inteligente.

COGNITIVE ROUTER
Routing por Intent + Domínio
Classifica queries por intent (query/action/report), detecta domínios (regulatory/financial/operational), e roteia para o agente mais qualificado. Learning loop que melhora com cada interação.
HYBRID RAG
ChromaDB + BM25 Retrieval
Busca híbrida: embeddings semânticos (Gemini) + busca lexical BM25. Corpus de 5,640 documentos com chunking otimizado (1024 tokens, overlap 128). Top-k=8 com reranking.
METACLAW PROXY
Skills Auto-Aprendidas
MetaClaw observa interações e cria skill candidates automaticamente. Skills com uso recorrente são promovidas. Skills não utilizadas são podadas. Self-learning loop.
AUTORESEARCH
Eval Loop (Karpathy Method)
paganini autoresearch roda N iterações otimizando parâmetros RAG contra um eval set gold. Auto-melhora qualidade sem intervenção humana.
RTK COMPRESSION
60-90% Token Savings
Proxy Rust que comprime output de comandos shell antes de alimentar o contexto LLM. Reduz custo operacional em 85%. exec_compressed() em todos os agentes.
DAEMONS
Monitoramento 24/7
Covenant Monitor (liquidez/subordinação/inadimplência), PDD Calculator (aging em tempo real), Reconciliation (conferência automática). Rodam como daemons persistentes.
