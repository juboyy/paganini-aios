1. Phase 1 — Foundation (Semanas 1-4)

Tornar o kernel multi-tenant e servível. MVP que roda 1 cliente real.
#DeliverableDescriçãoEsforçoDepsPrioridade
1.1 Gateway Multi-Tenant Entry point HTTP que recebe mensagens de qualquer canal, identifica o cliente (API key ou webhook secret), e roteia para o kernel. Rate limiting, auth, logging. Cloudflare Worker ou FastAPI. 5d — P0
1.2 Workspace Manager CRUD de workspaces. Cada cliente = 1 workspace (SOUL.md, memory/, skills/, config.yaml). Storage: R2 (cloud) ou filesystem (on-prem). Hydrate/dehydrate em <50ms. 5d — P0
1.3 Onboarding Engine Conversa aberta que coleta informações do cliente e gera SOUL.md + config.yaml automaticamente. Sem formulários, sem menus. O AIOS pergunta, o cliente responde. 3d 1.2 P0
1.4 LLM Proxy (MetaClaw v2) Proxy que roteia queries entre Flash/Sonnet/Opus baseado em complexidade. Cache layer (KV). Token counting. Cost tracking per workspace. BYOK support (cliente usa sua key). 4d 1.1 P0
1.5 Canal: Telegram Bot Bot Telegram standalone (não via OpenClaw). Webhook → Gateway → Kernel → resposta. Suporte a texto, imagens, documentos, voice notes. 3d 1.1 P1
1.6 Canal: Slack App Slack App com OAuth. Events API → Gateway. Slash commands, threads, file uploads. 3d 1.1 P1
1.7 Data Ingestor Pipeline genérico: CSV, PDF, API endpoints, CVM/B3 feeds. Extrai → chunka → embeda → armazena no workspace RAG index. Deve funcionar pra qualquer domínio. 5d 1.2 P0
1.8 RunPod Serverless Inference Endpoint funcional do Qwen 27B GPTQ. Resolver problema de GPU scarcity (flashboot, pre-warm, fallback provider). OpenAI-compatible API. 2d — P1
Gate de saída Phase 1: 1 cliente real usando o Paganini via Telegram ou Slack. Workspace isolado. Dados ingeridos. Agentes respondendo queries. Custo < R$50/mês de infra.

2. Phase 2 — Intelligence (Semanas 5-8)

Tornar o sistema inteligente: skills, evolution, e routing sofisticado.
#DeliverableDescriçãoEsforçoDepsPrioridade
2.1 Skill Registry Repositório central de skills. CRUD, versionamento, search. Cada skill = SKILL.md + scripts/ + tests/. API REST + CLI. Storage: R2 + Supabase metadata. 5d 1.2 P0
2.2 SkillRouter Implementar o paper arXiv:2603.22455. Encoder 0.6B + Reranker 0.6B. Indexa corpo completo da skill (não só nome/desc). 74% top-1 accuracy em 80K skills. Roda local (1.2B params). 7d 2.1 P0
2.3 Memento-Skills Engine Implementar evolução de skills. Cada skill tem performance_score. Reflection loop diário: identifica skills fracas → gera variantes → testa → promove melhores. Cross-client pattern detection anonimizado. 7d 2.1 P1
2.4 Guardrail Framework Generalizar os 6 gates FIDC para framework plugável. Cada domínio define seus guardrails via config. Gates: eligibility, concentration, covenant, compliance, risk, PLD/AML — todos configuráveis. 4d 1.3 P1
2.5 Alerting System Alerta pro-ativo via canal do cliente. Triggers configuráveis (threshold, schedule, event). Template de mensagem. Throttling (não enviar 100 alertas/dia). 3d 1.5, 1.6 P1
2.6 Report Generator PDF reports gerados sob demanda ou agendados. Templates por domínio. Gráficos, tabelas, análises. Enviados no canal do cliente. 4d 1.7 P1
2.7 Heartbeat Autônomo Cada workspace tem seu heartbeat loop. Checka dados, gera alertas, roda reflection. Cron distribuído (não centralizado). Billing-aware (não roda se plano não inclui). 3d 1.2, 2.5 P1
Gate de saída Phase 2: 5-10 clientes ativos. Skills instaláveis do Registry. Alertas funcionando. Reports PDF. SkillRouter roteando com precisão. Skill evolution rodando em background.

3. Phase 3 — Scale (Semanas 9-14)

Infra para 100+ clientes. Billing. Multi-canal. Segurança enterprise.
#DeliverableDescriçãoEsforçoDepsPrioridade
3.1 Dynamic Workers (Cloudflare) Migrar Gateway de FastAPI/EC2 para Cloudflare Workers. Zero idle cost. Auto-scale. KV para state, R2 para workspace, Durable Objects para sessions. 7d 1.1 P0
3.2 Billing System Stripe integration. Planos (Starter/Pro/Business/Enterprise/On-Prem). Usage tracking (queries/dia). Overage billing. Invoicing automático. Webhook handlers. 7d 1.4 P0
3.3 Dashboard Real-Time Dashboard web para cada cliente. Auth (email/password + SSO). Mostra: queries, custos, skills ativas, alertas, reports, performance. WebSocket para real-time. 10d 3.2 P1
3.4 Canal: Microsoft Teams Teams Bot via Azure Bot Service. Adaptive Cards para outputs ricos. OAuth para workspace linking. 5d 1.1 P1
3.5 API REST Pública API REST para integração com sistemas do cliente. OpenAPI spec. API keys. Rate limiting. Webhooks de eventos. SDK Python + TypeScript. 5d 1.1, 3.2 P1
3.6 Tenant Isolation & Security Anonymization gate no reflection loop. Workspace encryption at rest. Audit trail (quem acessou o quê). IAM policies. Pen test report. 7d 1.2, 3.1 P0
3.7 TUI Client Terminal UI (Rich/Textual). Conecta ao workspace via API. Chat, queries, alertas, skill management. Para devs e power users. 3d 3.5 P2
3.8 Landing Page & Marketing Site Site de vendas. Pricing page. Demo interativa. Blog (SEO). Formulário de contato enterprise. Analytics (PostHog/Plausible). 5d — P0
3.9 Domain Packs: Logística + Jurídico 2 packs novos para validar domain-agnostic: Logística (rastreamento, otimização rota, estoque) e Jurídico (análise contrato, prazo, jurisprudência). Cada um com 5-7 agentes + guardrails. 10d 2.4 P1
Gate de saída Phase 3: 50-100 clientes. Billing funcionando. 3+ canais. Dashboard real-time. API pública documentada. 3 domain packs (FIDC + Logística + Jurídico). Segurança auditada.

4. Phase 4 — Enterprise (Semanas 15-20)

Features enterprise: SSO, on-prem, SFT customizado, compliance.
#DeliverableDescriçãoEsforçoDepsPrioridade
4.1 SSO/SAML/OIDC Integração com Active Directory, Okta, Auth0. SCIM provisioning. Role-based access (admin, analyst, viewer). 7d 3.3 P1
4.2 On-Prem Installer Script/Helm chart que deploya o Paganini inteiro na infra do cliente. Suporte: bare metal (Mac Mini M4), Docker, K8s. Air-gap mode (zero internet). 10d 3.1 P1
4.3 Custom SFT Pipeline Pipeline pra fine-tunar Qwen pro domínio do cliente. Coleta dados do workspace → gera dataset → treina no RunPod → deploy modelo customizado. Tudo automatizado. 7d 1.8 P1
4.4 Compliance Engine Audit trail completo (queryable). Data retention policies. LGPD compliance (delete on request). SOC 2 prep. Exportar logs em formato regulatório. 7d 3.6 P1
4.5 BYOK Proxy Cliente configura suas API keys (OpenAI, Anthropic, Google). MetaClaw roteia usando a key do cliente. Zero custo LLM pra nós. Dashboard mostra consumo. 3d 1.4 P1
4.6 Hyperagent Meta-Loop Implementar paper FAIR (arXiv:2603.19461). Meta-improvement cross-domain. Sistema identifica padrões de sucesso em um domínio e transfere para outros. O Paganini no mês 12 é incomparavelmente melhor que no mês 1. 10d 2.3 P2
Gate de saída Phase 4: 1º cliente Enterprise (R$2.499/seat). SSO funcionando. 1 deploy on-prem. SFT customizado entregue. Compliance audit trail operacional.

5. Phase 5 — Moat (Semanas 21-26)

Construir defensibilidade: marketplace, rede, efeitos de escala.
#DeliverableDescriçãoEsforçoDepsPrioridade
5.1 Skill Marketplace Marketplace público de skills. Qualquer dev publica. Review automático (testes + security scan). Revenue share: 70% dev / 30% Paganini. Inspirado no TIG protocol (submit → benchmark → verify → reward). 10d 2.1, 2.2 P1
5.2 OpenClaw-RL Training Loop Framework RL que treina modelo a partir de conversas reais. Cada interação do cliente = datapoint. Modelo fica mais inteligente com uso. RLHF contínuo. 14d 4.3 P2
5.3 Partner API / White-Label API para parceiros embeddarem Paganini no produto deles. White-label: remove branding Paganini. Revenue share model. 7d 3.5 P2
5.4 Multi-Language Support Suporte a EN, ES, PT-BR nativo. Detecção automática. SOUL.md + skills traduzidos. LLM prompts multilíngues. 5d 1.3 P2
5.5 Mobile App (iOS/Android) App nativo para receber alertas push, consultar dados, e falar com o AIOS. React Native ou Flutter. Offline mode com sync. 14d 3.5 P2
