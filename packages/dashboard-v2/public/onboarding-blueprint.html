4. Anatomia do pacote

O que tem dentro do `pip install paganini-aios`.
KERNEL (fixo, não customizado)
moltis.py — LLM runtime (multi-provider)
metaclaw.py — skill routing + evolution
router.py — cognitive agent dispatch
engine.py — query pipeline orchestrator
pipeline.py — BMAD-CE execution
memory.py — RAG + persistent memory
guardrails/ — compliance gate pipeline
cli.py — paganini init / up / query / serve
GERADO NO ONBOARDING (customizado por cliente)
SOUL.md — comportamento (tom, regras, anti-patterns)
AGENTS.md — quais agentes, como orquestrar
agents/souls/*.md — SOUL de cada agente
MEMORY.md — contexto fundacional do cliente
HEARTBEAT.md — rotinas autônomas
guardrails/rules/ — regras específicas do domínio
integrations/ — conexões com ferramentas do cliente
SKILL REGISTRY (remoto, evolui continuamente)

O Registry é um repositório central curado pela Paganini. Quando o cliente faz paganini init, o sistema puxa as skills relevantes do Registry pro domínio dele. Quando skills evoluem (nossas ou de outros clientes), o cliente recebe updates via paganini update.

pld-scan-v3 concentration-v2 cvm-ingest covenant-monitor cedente-score mark-to-market dd-report regulatory-watch risk-scanner m&a-valuation esg-screening slack-alerts pdf-report-gen market-data-b3

5. Como o onboarding gera tudo

Não é template. É geração via LLM baseada nas respostas do cliente.
FLUXO INTERNO DO `paganini init`
1
Coleta — 7 perguntas abertas. Cliente descreve em linguagem natural: empresa, setor, dores, ferramentas, relatórios, preferência de LLM, equipe.
2
Classificação — Kernel analisa respostas → identifica: domínio (FIDC, M&A, Asset, etc.), complexidade (fundos, volume, regulação), integrações necessárias.
3
Domain Pack — Seleciona o pack mais próximo do Registry. Ex: "administração fiduciária FIDC" → finance-fidc pack (9 agents, 6 guardrails, 9 skills).
4
Geração — LLM gera SOUL.md, agent SOULs, HEARTBEAT.md customizados. Não são templates com variáveis. São documentos escritos PARA aquele cliente, referenciando as dores e processos que ele descreveu.
5
Integração — Configura conexões com as ferramentas do cliente (Slack webhook, API endpoints, email SMTP). Gera YAML de integração.
6
Ingestão — Se FIDC: puxa dados CVM Dados Abertos por CNPJ. Se outro domínio: cliente aponta fonte de dados (CSV, API, DB). Sistema ingere e indexa.
7
Validação — Roda um scan de teste com os dados ingeridos. Mostra resultados ao cliente. "Encontramos X problemas nos seus fundos." Primeiro valor imediato.

6. Exemplo: SOUL.md gerado para Vórtx DTVM

Gerado automaticamente pelo onboarding. Cada cliente recebe um diferente.
# SOUL.md — Paganini AIOS para Vórtx DTVM

## Identidade
Sou o sistema de inteligência operacional da Vórtx DTVM.
Administramos 47 FIDCs com PL total de R$8.2B.
Minha missão: zero multas CVM, zero surpresas de compliance.

## Comportamento
- Compliance é a prioridade #1. Na dúvida, bloqueia.
- Concentração acima de 18% → alerta imediato (antes do limite de 20%)
- PLD/AML: qualquer match OFAC/COAF → escala pra humano, nunca aprova sozinho
- Relatórios CVM: gera draft automático D-5, revisão humana D-3, envio D-1
- Tom: direto, técnico, sem jargão desnecessário
- Idioma: português formal (comunicação com CVM/cotistas)

## Rotinas autônomas
- 07:00: scan de concentração em todos os 47 fundos
- 09:00: ingestão de dados B3/BCB do dia anterior
- 14:00: PLD/AML scan em novas operações
- 17:00: mark-to-market + alerta se NAV divergir >0.5%
- Semanal: relatório consolidado → Slack #gestao

## O que nunca fazer
- Nunca aprovar operação que falhou num guardrail sem escalação humana
- Nunca enviar relatório CVM sem aprovação do diretor de compliance
- Nunca expor dados de um fundo para agentes de outro fundo

7. CLI do cliente

Tudo que o cliente pode fazer após o setup.
# Setup
$ paganini init — onboarding interativo
$ paganini up — liga o sistema (heartbeat, daemons)
$ paganini down — desliga

# Operação
$ paganini query "qual a concentração do FIDC Atlas?"
$ paganini scan --fund "FIDC Atlas Premium" — compliance scan
$ paganini report dd --fund "FIDC Atlas" --output pdf
$ paganini ingest --cnpj 12.345.678/0001-90 — ingere fundo novo

# Skills
$ paganini skills list — skills instaladas
$ paganini skills search "covenant" — busca no Registry
$ paganini skills install covenant-monitor-v2
$ paganini update — atualiza skills do Registry

# Evolução
$ paganini evolve — roda reflection loop (melhora skills que falharam)
$ paganini memory search "concentração" — busca na memória
$ paganini serve — sobe API REST (pra conectar frontend)
