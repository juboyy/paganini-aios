6. Estratégia de LLM — Nosso vs BYOK vs Local

Três caminhos, três perfis de cliente, três modelos de receita.

① Nosso Proxy (Starter/Pro/Business)

Como funciona: Cliente usa nosso proxy. MetaClaw roteia: Flash pra triagem, Sonnet pra análise, Opus pra decisão. Cliente não precisa de API key.

Nosso custo: R$0.10/query (média ponderada)

Receita: Incluída no assento + excedente

Margem LLM: 60-78% (spread entre custo e preço do plano)

Vantagem: Zero friction no onboarding. Cliente começa em 2 minutos.

Risco: Se pricing do provider sobe, margem comprime.

Flash 62% Sonnet 30% Opus 5% SFT 3%

② BYOK (Enterprise)

Como funciona: Cliente traz sua API key (OpenAI, Anthropic, Google). Nosso proxy roteia mas usa a key dele. Custo LLM = zero pra nós.

Nosso custo: R$0/query

Receita: Puro assento (R$2.499/mês)

Margem LLM: 100% (não pagamos LLM)

Vantagem: Margem máxima. Cliente controla seus custos.

Risco: Cliente vê quanto gasta no provider e questiona nosso preço.

Mitigação: valor está nos agentes, skills e evolução — não no LLM em si.

③ On-Prem / Local (On-Prem)

Como funciona: Qwen 27B SFT roda no hardware do cliente (Mac Mini M4 Pro 64GB ou GPU server). Zero dados saem da rede.

Nosso custo: R$0/query

Receita: Assento + setup + training fee

Margem LLM: 100%

Vantagem: Air-gap, compliance total, dados protegidos.

Risco: Performance do modelo local (14 tok/s no 27B Q4) vs cloud (60+ tok/s).

Qwen 27B SFT: 14 tok/s
Qwen 8B fast: 55 tok/s

Arquitetura: 8B local pra triagem (rápido) + 27B local pra análise (preciso). Sem cloud.

7. Deployment enterprise — 4 padrões

Como implementar numa empresa grande. Simples, adaptável, sem friction.

Padrão A — Channel-First (2 dias)

Para: Empresa que quer testar rápido. Mínimo friction.

Dia 1Bot adicionado ao Slack/Teams da empresa
Dia 1Onboarding conversacional com 1 stakeholder
Dia 1Workspace gerado, skills instaladas
Dia 2Dados ingeridos (CVM, CSV, API)
Dia 2Primeiro compliance scan + relatório

Esforço cliente: 30 min de conversa + share de dados

Padrão B — API Integration (1 semana)

Para: Empresa que quer integrar no sistema deles.

Dia 1-2Onboarding + workspace gerado
Dia 3-4API REST habilitada (paganini serve)
Dia 4-5Webhooks configurados (eventos → sistema do cliente)
Dia 5-6Dashboard embeddable fornecido
Dia 7Go-live + monitoramento

Esforço cliente: 1 dev pra integrar endpoints

Padrão C — Departmental Rollout (2 semanas)

Para: Empresa grande, rollout por departamento.

Semana 1Piloto com 1 departamento (5-10 assentos)
Semana 1SSO/SAML integrado com AD/Okta
Semana 2Skills customizadas pro workflow do depto
Semana 2Training session (1h) com usuários
Mês 2-3Expansão para outros departamentos

Esforço cliente: 1 champion interno + IT pra SSO

Padrão D — On-Prem (4 semanas)

Para: Banco, governo, regulador. Zero cloud.

Semana 1Hardware spec + aquisição (Mac Mini/GPU)
Semana 2Deploy do kernel + modelo SFT local
Semana 2Onboarding + workspace gerado
Semana 3Custom training com dados do cliente
Semana 3Guardrails calibrados
Semana 4Security audit + go-live

Esforço cliente: Infra team + security review + dados pra training

10. Resumo executivo

O modelo Paganini
Fábrica autônoma: Dynamic Workers servem N clientes com zero idle cost e zero headcount até 500 clientes.
5 planos: Starter (free/usage) → Professional (R$299/seat) → Business (R$999/seat) → Enterprise (R$2.499 BYOK) → On-Prem (R$4.999 isolated).
LLM strategy: Nosso proxy (margem 60-78%) para Starter-Business. BYOK (margem 98%) para Enterprise. Local SFT (margem 99%) para On-Prem.
Skill economy: Registry free como bait, Pro como upsell, Custom como R&D financiado pelo cliente.
Escala: 10K clientes = $800/mês de infra. Margem melhora com escala.
Enterprise deploy: 4 padrões — Channel-First (2 dias) a On-Prem (4 semanas). Adaptável ao nível de paranoia do CISO.
ARR projetado: R$1.14M (ano 1) → R$4.77M (ano 2) com 2-3 pessoas.
Hyperagent loop: Sistema fica mais inteligente com cada cliente. Meta-improvement transfere entre domínios. O produto no mês 24 é incomparavelmente melhor que no mês 1.
