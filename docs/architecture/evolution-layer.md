# PAGANINI AIOS — Evolution Layer (MetaClaw Integration)

## Overview

MetaClaw (`aiming-lab/MetaClaw`) is an OpenAI-compatible learning proxy that sits
between the runtime (Moltis) and the LLM provider. It intercepts agent interactions,
detects patterns, generates skills automatically, and injects them into future calls.

The agent improves with every interaction. No fine-tuning. No GPU. No retraining.

## Architecture Position

```
User → Moltis (runtime) → MetaClaw (learning proxy) → LLM Provider (BYOK)
```

MetaClaw is TRANSPARENT to Moltis. Moltis calls what it thinks is the LLM provider.
MetaClaw intercepts, enriches with learned skills, forwards, and learns from the response.

## Layer Separation

| Concern | Moltis (Execution) | MetaClaw (Evolution) |
|---------|-------------------|---------------------|
| Scope | What the agent DOES | What the agent KNOWS |
| Skills | Tool definitions (search, calc, fetch) | Behavioral patterns ("when X, context Y is relevant") |
| Memory | Operational state (portfolio, covenants) | Interaction patterns (effective queries, good responses) |
| Sessions | User sessions (chat, operations) | Learning sessions (what improved, what didn't) |
| Improvement | Static (config changes) | Dynamic (continuous from interactions) |

## Three Improvement Loops

```
Level 1: AUTORESEARCH → optimizes RETRIEVAL
  (chunking, embedding, ranking, RRF weights)
  Frequency: continuous background loop
  Metric: eval_score (precision + recall + faithfulness)

Level 2: METACLAW → optimizes BEHAVIOR
  (skill generation, context injection, pattern detection)
  Frequency: every interaction
  Metric: response quality + user satisfaction signals

Level 3: MEMORY REFLECTION → optimizes KNOWLEDGE
  (graph evolution, ontology expansion, relation discovery)
  Frequency: daily daemon
  Metric: knowledge graph coverage + contradiction rate
```

## Configuration

```yaml
# config.yaml addition
metaclaw:
  enabled: true
  mode: proxy                              # proxy | embedded
  endpoint: "http://localhost:8100"         # MetaClaw proxy address
  isolation: per-instance                   # per-instance | per-fund | shared
  
  skill_validation:
    enabled: true                          # Validate skills before activation
    validator: guardrails                  # guardrails | llm-judge | manual
    reject_on_contradiction: true          # Reject skills that contradict corpus
    max_active_skills: 500                 # Prevent unbounded skill accumulation
  
  skill_review:
    enabled: true
    interval: 7d                           # Weekly skill review
    auto_prune_below_score: 0.3            # Remove low-quality skills
    drift_detection: true                  # Alert if eval score degrades after new skills
  
  audit:
    log_injected_skills: true              # Every response traces which skills were active
    skill_changelog: true                  # Track skill creation/modification/deletion
    exportable: true                       # Skills exportable for regulatory review
```

## Moltis Integration

MetaClaw acts as the LLM endpoint for Moltis:

```yaml
# moltis provider config
providers:
  default:
    type: openai-compatible
    base_url: "http://localhost:8100/v1"    # MetaClaw proxy
    api_key: "${CLIENT_API_KEY}"           # Passed through to real provider
    # Moltis thinks it's talking to OpenAI. MetaClaw intercepts transparently.
```

## Financial Domain Guardrails for Skills

Skills auto-generated by MetaClaw pass through validation before activation:

```
Skill Generated → Validation Pipeline:
  1. Corpus Check: Does skill contradict any document in corpus?
  2. Ontology Check: Are entities/relations in skill valid per ontology?
  3. Regulatory Check: Does skill comply with CVM 175 / fund regulations?
  4. Conflict Check: Does skill contradict any existing active skill?
  5. Quality Check: Is skill specific enough to be useful? (no generic platitudes)

  ALL PASS → skill activated
  ANY FAIL → skill quarantined for human review
```

## Data Isolation

**Critical for multi-tenant deployment:**

- Each client/instance runs its OWN MetaClaw process
- Skills learned from Client A NEVER appear in Client B's context
- MetaClaw data directory is per-client: `runtime/metaclaw/{client_id}/`
- No shared skill store across clients
- Skills are part of the client's intellectual property

## Skill Lifecycle

```
1. DETECTION: MetaClaw observes interaction pattern
2. GENERATION: LLM generates skill definition (trigger + context + template)
3. VALIDATION: Guardrail pipeline checks for conflicts/contradictions
4. ACTIVATION: Skill added to active pool
5. MONITORING: Track skill usage frequency + quality impact
6. REINFORCEMENT: Frequently used skills get higher injection priority
7. PRUNING: Unused or low-quality skills pruned in weekly review
8. AUDIT: Full lifecycle logged for regulatory compliance
```

## Latency Impact

| Component | Latency | Notes |
|-----------|---------|-------|
| Moltis → MetaClaw | ~5ms | Local network (same host) |
| MetaClaw skill injection | ~15ms | Pattern matching + context assembly |
| MetaClaw → LLM Provider | ~0ms | Direct forward (no modification) |
| **Total overhead** | **~20ms** | Negligible vs 1-5s LLM inference |

## Dependency Management

MetaClaw is forked into the PAGANINI monorepo to ensure:
- No upstream breaking changes affect production
- Financial-specific modifications maintained internally
- Contributions pushed upstream when appropriate
- Full control over skill validation logic

```
paganini/
└── vendor/
    └── metaclaw/        # Controlled fork of aiming-lab/MetaClaw
```

## Risk Register

| Risk | Severity | Mitigation |
|------|----------|------------|
| Skill contradicts regulation | HIGH | Validation pipeline + corpus check |
| Skill drift changes agent behavior | MEDIUM | Weekly review + eval regression detection |
| Data leakage between clients | HIGH | Per-instance isolation, never shared |
| MetaClaw project abandoned upstream | LOW | Controlled fork in monorepo |
| Latency degradation | LOW | Local deployment, ~20ms overhead |
| Skill explosion (too many skills) | MEDIUM | Max cap (500) + pruning daemon |
| Auditor can't trace skill influence | HIGH | Full skill injection logging in traces |
