1 — Mapa de Disrupção

O que estamos substituindo

O Bloomberg Terminal custa $24.000/ano por usuário. As alternativas (Refinitiv, FactSet) cobram $15.000-20.000. Nenhum deles entende português, FIDC, ou regulação brasileira. Nenhum deles usa AI agentic.

Modelo Incumbente (Bloomberg)
  • $24K/ano por terminal
  • Hardware proprietário (teclado Bloomberg)
  • Dados estruturados, queries manuais
  • Zero AI nativa — chat bolt-on
  • Regulação brasileira como cidadão de segunda classe
  • Interface de 1990 — curva de aprendizado de meses
  • Lock-in total: dados não exportam, API cara
Paganini Terminal
  • R$2.990/mês — 85% mais barato
  • Web-native, qualquer dispositivo
  • AI agentic: pergunta em linguagem natural
  • 9 agentes FIDC especializados + AlphaEar
  • Regulação brasileira nativa (CVM, ANBIMA, BACEN)
  • Interface conversacional — zero curva de aprendizado
  • Open data: RAG + knowledge graph + API aberta
"O terminal não é um dashboard que você olha. É um sistema que olha pra você — e te avisa antes que você precise perguntar." Princípio fundador do Paganini Terminal

3 — Módulos Core

8 pilares do Terminal

Cada módulo é independente, testável, e plugável via pack architecture. O FIDC é o primeiro pack — a arquitetura suporta qualquer vertical financeiro.

Core Engine

Moltis Kernel + Cognitive Router

O cérebro. Recebe input em linguagem natural, classifica complexidade (simple/moderate/complex), roteia pro modelo certo (7B ou 27B), orquestra agentes, e monta a resposta. Compactação automática de contexto quando histórico passa de 24K tokens.

  • CognitiveRouter com 3 tiers de complexidade
  • Moltis runtime gerencia estado e memória
  • MetaClaw proxy para tool calling
Data Pipeline

Market Data Ingestion

Feeds em tempo real de B3, CVM, ANBIMA, CETIP. Normalização, dedup, e indexação automática no RAG pipeline.

  • alphaear-stock para cotações
  • alphaear-news para 10+ fontes
  • Polling configurable (1min–1hr)
AI / RAG

Hybrid RAG Pipeline

6 técnicas avançadas: contextual headers, multi-query rewriting, metadata filtering, compression, reranking, GraphRAG multi-hop.

  • 27 famílias de sinônimos FIDC
  • Dense + sparse + RRF fusion
  • Pack-driven: domain-agnostic core
AI / Agents

Fleet de Agentes

9 agentes FIDC especializados + 9 AlphaEar skills. Cada agente tem domínio, guardrails, e compliance built-in.

  • Gestor, compliance, pricing, IR
  • Due diligence, custódia, reporting
  • FinBERT sentiment + Kronos prediction
Decision

6 Guardrail Gates

Pipeline de validação: Eligibilidade → Concentração → Covenant → PLD/AML → Compliance → Risco. Nenhuma decisão passa sem cruzar os 6 gates.

  • Score automático por cedente
  • Limites regulatórios embutidos
  • Audit trail completo
Intelligence

Signal Tracker + Predictor

Monitora sinais de investimento em tempo real. Kronos faz time-series forecast com ajuste por sentiment de notícias. Cada sinal evolui: strengthen / weaken / falsify.

  • alphaear-signal-tracker ativo
  • alphaear-predictor (Kronos)
  • Logic chain visualization
Broadcast

Multi-Channel Delivery

O terminal não é um único app. É broadcast. Mesmo dado, mesma inteligência, 6 superfícies: Dashboard web, Slack, Telegram, API REST, Email digest, Webhooks.

  • Conversacional: Slack, Telegram
  • Visual: Dashboard terminal
  • Programático: API + Webhooks
Infra

Sentrux Quality Gate

Sensor arquitetural em Rust monitorando 720 arquivos. 6 layers definidos com dependency inversion enforced. Quality score: 5690.

  • 17 violações mapeadas (tech debt)
  • MCP server ativo para subagentes
  • CI/CD gate via sentrux check
4 — Data Pipeline

Fontes de dados e ingestão

O terminal é tão bom quanto seus dados. A ingestão precisa ser automatizada, incremental, e multi-fonte.

Fonte Tipo Frequência Formato Status
CVM Regulamentos, instruções, pareceres Diário (incremental) PDF → Markdown → RAG chunks A construir
ANBIMA Diretrizes, classificações, benchmarks Semanal PDF/HTML → Markdown A construir
B3/CETIP Cotações, volumes, títulos registrados Tempo real (5min delay) API → JSON → TimeSeries A construir
News (AlphaEar) Notícias financeiras, Polymarket Real-time (polling 5min) API → Sentiment → RAG Skill pronta
Dados do cliente Carteiras, NAV, PDD, cotas Upload ou API do ERP CSV/Excel/API → Schema validated A construir
Regulamentos FIDC Documentos do fundo específico On-demand (upload) PDF → OCR → Chunks → RAG RAG pronto, ingestão parcial
Princípio de ingestão

Todo documento entra pelo RAG pipeline com contextual headers + metadata extraction automática. O knowledge graph atualiza entidades em tempo real. Nenhum dado fica "solto" — tudo indexado, linkado, e searchable.

6 — Broadcast Engine

Delivery multi-canal

O Paganini não é um app. É um broadcast. Cada canal recebe o mesmo dado com formatação otimizada para a superfície.

Canal Principal

Dashboard Terminal (Web)

Interface estilo Bloomberg — painéis draggable, real-time ticker, gráficos interativos, chat AI integrado. O "monitor" da mesa de operações.

  • Next.js + TailwindCSS
  • WebSocket para real-time
  • Painéis configuráveis por usuário
  • Dark mode nativo (mesa de trading)
Conversacional

Slack + Telegram + Teams

Chat direto com os agentes. Pergunta em português, recebe análise com fontes. Alertas proativos de compliance, vencimento de cotas, limites de concentração.

  • Slack bot (já conectado)
  • Telegram bot (canal atual)
  • Microsoft Teams (roadmap)
Programático

REST API + Webhooks

Para integração com ERPs, sistemas de custódia, e plataformas de gestão. OpenAPI spec, rate limiting, autenticação por API key.

  • Endpoint /v1/query (OpenAI-compat)
  • Webhooks para eventos (alerta, NAV, PDD)
  • SDK Python para automação
Digest

Email + Reports

Digest diário automático: posição consolidada, alertas do dia, notícias relevantes, compliance status. Report mensal para cotistas gerado automaticamente.

  • alphaear-reporter para geração
  • Templates HTML responsivos
  • Scheduling configurável

9 — Métricas de Sucesso

O que medir

Produto

Qualidade RAG

  • Accuracy em 50 queries de benchmark FIDC
  • Recall: % de fontes relevantes recuperadas
  • Precision: % de chunks usados que eram relevantes
  • Target: >85% accuracy, >70% recall
Performance

Latência e Throughput

  • First token: <2s (API) / <5s (self-hosted)
  • Full response: <15s (complexa)
  • Concurrent users: 50+ por instância
  • Uptime: 99.5% (SLA enterprise)
Negócio

Revenue e Adoção

  • 5 clientes na Fase 3 (R$14.950/mês)
  • Daily active users per client: >3
  • Queries/dia/cliente: >15
  • Margem: >98% (infra < 2% da receita)
Arquitetura

Saúde do Sistema

  • Sentrux quality score: >6500 (atual: 5690)
  • Violações Sentrux: 0 (atual: 17)
  • Test coverage: >70%
  • Zero god files (max 200 LOC/function)

Risco Impacto Probabilidade Mitigação
Alucinação em dados regulatórios Crítico Alta RAG com citação obrigatória de fontes + 6 guardrail gates + validação humana em decisões críticas
Cold start inaceitável (83s) Alto Média Phase 1 via API (zero cold start). Phase 2+: keep-alive ou GPU dedicada
Dados CVM/ANBIMA difíceis de extrair Médio Alta PDF pipeline robusto com OCR fallback. Começar com documentos públicos mais estruturados
Modelo mistura chinês em respostas Médio Média System prompt reforçado + max_tokens ≥ 300 + fine-tune com dados PT-BR
Compliance: uso de AI em decisões financeiras Crítico Baixa Disclaimer explícito: "assistente, não consultor". Audit trail. Guardrails impedem ações diretas
Dependência do Together AI na Phase 1 Médio Baixa Interface OpenAI-compatible permite trocar provider em 1 config change. Modal como fallback testado
