Princípio

Open Source = Motor (adoção, contribuição, confiança) → Pago = Combustível (corpus, skills, agentes, templates). O framework roda 100% funcional com dados de exemplo. O valor real está no corpus regulatório curado (164 docs FIDC), skills treinadas, e agentes especializados.

3 — Detalhamento dos Itens Pagos
guardrails.py — Regras CVM 175 embutidas

Contém limites regulatórios hardcoded: concentração por cedente (10%, 20%), thresholds de elegibilidade, regras PLD/AML (PEP, sanções), limites de risco.

Solução: Separar engine (open) de rules (pack). O engine recebe regras como rules.yaml; o pack FIDC fornece o arquivo com os valores reais da CVM.

agents/souls/*.md — 9 SOULs FIDC especializados

Cada SOUL define comportamento, heurísticas regulatórias, domínios e keywords curadas por papel: Gestor, Administrador, Custodiante, Compliance, Reporting, Due Diligence, Regulatory Watch, Investor Relations, Pricing.

Solução: No repo aberto: _template.md + sample_agent.md. Os 9 SOULs reais vão em packs/fidc/souls/.

templates/reports/*.qmd — 5 templates regulatórios

Informe mensal (ANBIMA), CADOC 3040 (BACEN), PDD/IFRS9, Risk report, Covenant report. Cada template é produto de consultoria regulatória curada.

Solução: No repo aberto: _template.qmd genérico. 5 templates reais vão em packs/fidc/templates/.

daemons.py — 3 handlers com lógica FIDC

covenant_monitor (inadimplência/subordinação/concentração), pdd_calculator (IFRS9 staging: S1=0.5%, S2=5%, S3=20%), risk_scanner (VaR 99%, diversificação, liquidez). ~300 linhas de lógica de domínio.

Solução: Framework (scheduler, registry, logging) fica open. 3 handlers extraídos para packs/fidc/daemons/.

data/corpus/ — 164 docs regulatórios (já gitignored)

6.4MB de corpus curado: CVM 175 (57 artigos), 300 dores de mercado, IFRS9/PDD/COFIs, 20+ tipos de FIDC, API specs, 80 diferenciais competitivos. Já está no .gitignore. Entregue via paganini install fidc-*.

6 — Plano de Separação (Quando Abrir o Repo)

Extrair SOULs para packs/fidc/souls/
Mover 9 arquivos .md de packages/agents/souls/. Criar _template.md + sample_agent.md no lugar.

Extrair templates para packs/fidc/templates/
Mover 5 arquivos .qmd de templates/reports/. Criar _template.qmd genérico.

Refatorar guardrails.py → engine + rules.yaml
Engine genérico carrega regras de arquivo YAML. Pack FIDC fornece rules.yaml com valores CVM.

Extrair handlers FIDC de daemons.py
Separar covenant_monitor, pdd_calculator, risk_scanner (~300 linhas) em packs/fidc/daemons/. Framework loader descobre handlers do pack.

Reduzir eval_questions.jsonl
Manter 5 perguntas genéricas no repo. 20 completas vão em packs/fidc/eval/.

Adicionar markers PAGANINI-PAID
Marcar blocos de IP nos arquivos mistos com comentários especiais para extração automatizada via script.

Gitignore packs/ + scrub history
git filter-repo para remover SOULs e templates do histórico. Rodar trufflehog + gitleaks antes de abrir.

Criar data/sample/ com dados sintéticos
sample-regulation.md, sample-fund.json, sample-cedentes.json — framework funcional sem pack pago.
